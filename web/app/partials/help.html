CPA Secrity

Define Scheme

Before starting the proof, the types of the message variable and the random variables have to be given. Additionally the cipher has to be defined and all employed hash functions and trapdoor permutations have to be declared.

It is also possible to load one of the predefined schemes.

Proof of Scheme

The following rules can be applied to a goal:

  • fail1 (H) (r1) where H is a hash function symbol and r1is the name of a random variable that does not occur in the proof yet.
  • fail2 (H) (r1) where H is a hash function symbol and r1is the name of a random variable that does not occur in the proof yet.
  • opt (r) (e) where r is random variable that occurs in the goal and e is an expressions. The rule replaces r by r + e.
  • norm. The rule replaces all terms with their normal-forms with respect to =E.
  • merge (r1) (r2) (r)
  • split (r) (r1) (r2)
  • perm (r) (f). NOTE: not implemented yet.
  • ind
  • ow
  • rnd
By clicking on x, the corresponding subproof will be removed and the rule application will be replaced with Admit.

CPA Proof for OAEP based on partial-domain one-wayness: 

fail1 (G) (r1);
  opt (r1) (m_b | (0^{k2}));
  norm;
  ind;
opt (r1) (m_b | (0^{k2}));
norm;
fail1 (H) (r2);
  opt (r2) (r);
  norm;
  rnd ( * ) ( r );
opt (r) (r2);
norm;

norm;
